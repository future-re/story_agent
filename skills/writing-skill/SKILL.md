# Writing Skill

## 目标

在约束明确的前提下，稳定产出可发布的中文叙事内容，并保持人物一致性、剧情连贯性和风格统一。

## 工作流

1. 明确任务类型：`新写` / `续写` / `润色` / `改写` / `扩写` / `缩写`。
2. 提取硬约束：体裁、视角、人称、字数、禁用项、目标读者、语气。
3. 建立最小写作简报：`主题`、`冲突`、`角色目标`、`本段目标`、`结尾钩子`。
4. 先给结构后给正文：
   - 结构：3-6 条场景/段落计划。
   - 正文：按结构逐段生成，避免空泛总结。
5. 完成后自检并修订，再输出终稿。

## 续写规则

1. 直接承接上文最后一个动作或情绪，不跳时间线。
2. 保持角色口吻与知识边界，不让角色说出其未知信息。
3. 新角色初登场时先建立识别信息（外观、动机、关系状态）。
4. 章节结尾必须留下推进点：新问题、反转、决定或风险升级。

## 润色规则

1. 保持原剧情不变，优先改语言、节奏、画面感和对白辨识度。
2. 删除重复表达和解释性句子，保留动作与细节承载信息。
3. 统一术语、人称、时态（中文语境下统一叙述口径）。
4. 在不改剧情前提下增强“冲突-反应-后果”链条。

## 输出模板

先输出简报，再输出正文：

```markdown
【写作简报】
- 任务类型：
- 体裁/风格：
- 本段目标：
- 关键冲突：
- 角色弧线：
- 结尾钩子：

【正文】
（直接给可用文本）
```

## 质量检查清单

1. 开头是否在 3 段内进入事件。
2. 每个场景是否存在明确目标与阻力。
3. 对话是否能区分角色（词汇、语气、态度）。
4. 是否存在“告诉而不展示”的冗长说明。
5. 结尾是否有可驱动下一段的悬念或决策。

## 失败回退策略

1. 约束不足时，先给 2-3 个简短方向并继续写默认方向。
2. 信息冲突时，优先服从用户最新指令并显式说明采用的版本。
3. 长文本上下文不足时，先总结“已知事实表”再继续正文。
